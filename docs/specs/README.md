# 設計ドキュメント体系

このフォルダはポートフォリオサイトの要件定義・設計ドキュメントを管理する。

## フォルダ構成

```
docs/specs/
├── README.md              ← このファイル（ドキュメント体系ガイド）
├── overall/               ← 全体設計（プロジェクト全体の方針）
│   └── overall-design.md
├── pages/                 ← 各ページの詳細設計（随時追加）
│   ├── top.md             ← TOPページ
│   ├── works.md           ← 実績一覧ページ
│   └── service.md         ← サービスページ
└── features/              ← 各機能の要件定義（随時追加）
    ├── contact-form.md    ← お問い合わせフォーム
    ├── navigation.md      ← ナビゲーション
    └── responsive.md      ← レスポンシブ対応
```

## ブランチ運用

| 対象 | ブランチ | 例 |
|------|---------|----|
| 全体設計・基盤 | `main` | — |
| 各ページ | `feature/<page>` | `feature/top-page` |
| 各機能 | `feature/<feature>` | `feature/contact-form` |

1. 作業対象ごとに `main` から `feature/xxx` ブランチを切る
2. ドキュメント作成 → 実装 → 動作確認
3. 完成したら `main` にマージ
