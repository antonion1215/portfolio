# TOPページ 詳細設計（index.html）

作成日: 2026-06-22
ブランチ: `feature/top-page`

## 概要

ポートフォリオの入口となるページ。全要素を簡潔に要約し、興味を引いて詳細ページ（works / service）やお問い合わせへ誘導する。デザインはミニマル・洗練（オフホワイト × ブラック）。モバイルファーストで実装し、スマホではナビをハンバーガーメニュー化する。

## セクション構成（上から順に）

### 0. 共通ヘッダー（全ページ共通）

- 左: サイトロゴ（`Yusuke Yamahara` テキスト）
- 右: ナビ `ABOUT / SKILLS / WORKS / SERVICE / CONTACT`
- スマホ: ナビをハンバーガーメニュー化（タップで開閉、開くと黒背景の全画面メニュー）
- ナビ項目はすべてTOP内のページ内アンカー（`#about` `#skills` `#works` `#service` `#contact`）。詳細ページ（works.html / service.html）への遷移は各セクション内の「もっと見る / 詳しく見る」ボタンで行う

### 1. Hero

- ラベル: `WEB CREATOR`
- キャッチコピー（姿勢軸）: **「丁寧なものづくりで、想いをカタチに。」**
- サブ: `フリーランス Web制作 / 山原 佑介`
- CTAボタン2つ: 「実績を見る」（works.html へ）／「お問い合わせ」（#contact へ）
- 区切りの短い横線をアクセントに

### 2. About

- ラベル: `ABOUT` ／ 見出し: `About Me`
- 丸型のプロフィール写真（または代替ビジュアル）＋ 自己紹介テキスト
- 内容: Web制作への姿勢・これまでの経緯・大切にしている価値観を簡潔に（テキストは後で差し込み）
- レイアウト: スマホ縦積み → PC横並び

### 3. Skills（できること）

- ラベル: `SKILLS` ／ 見出し: `できること`
- **要件単位**の7項目をカード形式で表示:

| 項目 | 説明 |
|------|------|
| Webサイト制作 | コーポレート・ポートフォリオ等のサイト制作 |
| LP制作 | 成果につながるランディングページ |
| SEO対策 | 検索で見つけてもらう内部最適化 |
| レスポンシブ対応 | スマホ・PCに最適化した表示 |
| WordPress構築 | 更新しやすいCMSの導入 |
| 機能開発（AWS） | データ処理・認証などの機能実装 |
| AI活用開発 | AIを使った開発サポート・効率化 |

- 使用技術（HTML/CSS/JS等）は各カードの説明文や About 内で軽く触れる
- レイアウト: スマホ2列 → PC3列

### 4. Works（抜粋）

- ラベル: `WORKS` ／ 見出し: `制作実績`
- 実績カードを並べる（**現時点では実績ゼロ → 「準備中」プレースホルダを3枠表示**）
- 末尾に「もっと見る →」リンク（works.html へ）
- レイアウト: スマホ1列 → PC3列

### 5. Service（概要）

- ラベル: `SERVICE` ／ 見出し: `サービス`
- サービス概要を簡潔に記載（**内容は今後決定 → プレースホルダ**）
- 末尾に「詳しく見る →」リンク（service.html へ）

### 6. Contact

- ラベル: `CONTACT` ／ 見出し: `お問い合わせ`
- 背景は黒（`#1c1917`）で締めのセクションとして強調
- フォーム項目: お名前 / メールアドレス / お問い合わせ内容 + 送信ボタン
- **送信機能は未実装（UIのみ）**。将来 Formspree 等（方式A）を差し込めるよう、`<form>` のマークアップ（`label` / `input` / `textarea` / `required` 属性）は正しく組む
- 現時点の挙動: 送信ボタン押下では実送信せず、フォーム下に「※お問い合わせ機能は準備中です」の注記を表示する（`<form>` の `action` は未設定）

### 7. フッター（全ページ共通）

- コピーライト `© 2026 Yusuke Yamahara`

## レスポンシブ方針

- モバイルファースト（スマホ基準、PCは `@media screen and (min-width: 1080px)` で拡張）
- ナビ: スマホはハンバーガーメニュー（`js_` フックで開閉トグル、`.is-active`）
- Skills: スマホ2列 → PC3列
- Works: スマホ1列 → PC3列
- About: スマホ縦積み → PC横並び
- スマホの左右余白: `padding: 0 16px;`

## 使用クラス・命名（ワークスペース規約準拠）

- レイアウト: `l_`（例: `l_container`, `l_contents`）
- モジュール: `m_`（例: `m_btn`, `m_section_title`, `m_skill-card`）
- JSフック: `js_`（例: `js_hamburger`, `js_nav`）
- トグル状態: `.is-active`
- 修飾クラスはブロック名を冠する（例: `m_btn__cta`）
