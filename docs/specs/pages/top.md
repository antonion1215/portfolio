## 1. 概要

Web制作サービスの営業用LP。訪問者（中小企業経営者・個人事業主・飲食店経営者）が
**「自分の悩みを分かってくれている」→「この人なら解決できそう」→「この価格なら相談してみよう」** と段階的に納得し、無料相談フォームに到達する1ページ構成。

- 設計原則: 各セクションは必ずキャンバスのペイン/ゲインに対応する（対応がないセクションは置かない）
- ゴール行動: 無料相談フォームの送信
- CTA は「無料相談」で統一する（「お問い合わせ」は使わない — 心理的ハードルを下げるため）

| 「WEB CREATOR」ラベル・「丁寧なものづくりで、想いをカタチに。」コピー | 自分主語・抽象的。誰向けか10秒で伝わらない |
| works.html / service.html への導線 | ページ廃止（overall-design.md §7） |

### 0. 共通ヘッダー

- 左: ロゴ（**暫定: `Yusuke Yamahara`**。屋号決定後に差し替え → 残タスク T-1）
- 右ナビ: `お悩み / サービス / 料金 / 制作の流れ / 実績 / FAQ`（ページ内アンカー）
- **ナビ末尾に常設CTAボタン「無料相談」**（`#contact` へ。アクセントカラー）
- スマホ: ハンバーガーメニュー（v1.0 実装を流用）。メニュー内にも「無料相談」ボタンを置く

### 1. Hero

**目的**: 10秒で「誰の・何のサービスで・いくらからか」を伝え、無料相談へ誘導する。

コピー（3行コピー構造。実装時はこれを正とし、変更する場合は本ドキュメントを先に更新する）:

```
（Insight刺し）  「制作会社は高い」を、あきらめない。
（Promise）      IT企業で上流工程を担う現役エンジニアが、
                要件整理から公開までまるごと。コーポレートサイト30万円から。
（Action）      [ 無料で相談する → ]   [ 実績を見る ]
```

- CTA主ボタン: 「無料で相談する」（`#contact`、アクセントカラー・塗り）
- CTA副ボタン: 「実績を見る」（`#works`、アウトライン）
- 補足ラベル（コピー下に小さく）: 「平日夜・土日も打ち合わせ可能／お見積り無料」
- **ターゲット分岐バナー（Hero直下）**: 「飲食店・個人事業主の方」「企業サイトをご検討の方」の2カード。それぞれ `#service` 内の該当ブロックへアンカー

### 2. こんなお悩みありませんか（Pain）

**目的**: 訪問者に「自分のことだ」と思わせ、読み進める理由を作る。

- 悩みカード4枚（チェックマーク付きリスト形式でも可）:
  1. 「制作会社に見積りを取ったら、**予算を大きく超えていた**」
  2. 「何をどう頼めばいいのか、**そもそも分からない**」
  3. 「日中は仕事や店で手一杯。**打ち合わせの時間が取れない**」
  4. 「以前業者に頼んだが、**納品後に相談先がなくなった**」
- セクション末尾に橋渡し文: 「そのお悩み、すべて仕組みで解決できます。」→ 次セクションへ

### 3. 選ばれる理由（Reason / RTB）

**目的**: ペインリリーバー5点を「信じられる理由」とセットで提示する。番号付きカード5枚。

| # | 見出し | 本文の要点 |
|---|--------|-----------|
| 1 | 現役ITエンジニアが直接担当 | IT企業で要件定義・デザイン設計・AWS運用を日常業務として担当。その品質を個人価格で提供 |
| 2 | 要件整理からお任せOK | 「何を頼めばいいか分からない」状態からで大丈夫。ヒアリングから要件に落とすのが本業 |
| 3 | 制作会社より安い、理由がある | 営業・ディレクター等の間接コストがない1人体制。同じ品質を30万円〜で |
| 4 | 平日夜・土日に打ち合わせ | 経営者・店主の生活時間に合わせて対応。営業時間を潰さない |
| 5 | 納期と進捗を隠さない | 契約前に正直な納期を提示し、制作中は週次で進捗報告。「今どうなってる？」を無くす |

- 注意: ここは「私の強み」ではなく「あなたの不安がこう消える」の順で書く（表現ルール §10-1）

### 4. サービス内容（Service）

**目的**: 顧客ジョブ単位で「頼めること」を提示する。カード4枚。SEO・レスポンシブ・WordPressは各カードの説明文に「標準対応」として吸収し、独立カードにしない。

| カード | 対象 | 説明の要点 |
|--------|------|-----------|
| コーポレートサイト制作 | 中小企業 | 会社の信用をつくるサイト。スマホ対応・SEO内部対策・WordPressでの自社更新まで標準対応 |
| 採用サイト制作 | 中小企業 | 求人媒体に頼らない自社採用の入口。求職者目線の構成設計から |
| 店舗サイト制作 | 飲食店・店舗 | メニュー・アクセス・予約導線を整理し、来店につなげる |
| 機能開発（予約・フォーム・自動化） | 全対象 | 予約受付・お問い合わせ自動通知など、サイトに業務の仕組みを追加（AWS） |

- 各カードに「30万円〜」等の参考価格は書かず、§5 料金セクションへ誘導する（価格情報の一元管理）

### 5. 料金（Price）

**目的**: 「高い・不透明」ペインの解消。積み上げ式であることを構造で見せる。

- 基本価格表:

| 項目 | 価格 | 含まれるもの |
|------|------|------------|
| コーポレートサイト制作 | **300,000円〜** | ヒアリング・要件整理／デザイン／実装／スマホ対応／SEO内部対策／サーバー公開 |
| 採用サイト・店舗サイト | 個別見積（同水準の積算基準） | 同上 |
| 機能追加 | **+50,000円〜/件** | 予約フォーム・自動通知など1機能ごと |
| WordPress構築 | オプション | 自社更新できるCMS化 |

- 必須注記（3点セット）:
  1. 「お見積りは無料です」
  2. 「追加費用は**着手前に必ず事前提示**します。事後請求はありません」
  3. 「ページ数・機能により変動します。まずは概算をお出しします」
- セクション内CTA: 「無料で見積りを依頼する →」（`#contact`）

### 6. 制作の流れ（Flow)

**目的**: 「納期が読めない・放置される」不安の解消。5ステップの横並び（スマホ縦積み）。

| Step | 名称 | 説明 |
|------|------|------|
| 1 | 無料相談 | フォームから連絡 → 平日夜・土日にオンラインでヒアリング |
| 2 | 要件整理・お見積り | 聞いた内容を要件と概算見積りに整理して提示。**納期もここで確定** |
| 3 | デザイン | 構成案 → デザイン案。確認しながら進行 |
| 4 | 実装 | コーディング・機能開発。**週次で進捗を報告** |
| 5 | 公開・お引き渡し | サーバーアップ・動作確認。更新方法もレクチャー |

- 補足文: 「副業として稼働しているため、制作会社より納期に余裕をいただく場合があります。その分、**納期は最初に正直にお伝えし、途中経過を毎週共有します**。」（正直さの明文化 — 表現ルール §10-4）

### 7. 制作実績（Works）

**目的**: 品質の証拠。実物のみ掲載し、空き枠（「準備中」）は置かない。

- カード構成: サムネイル／種別タグ／タイトル／一言説明／「サイトを見る」リンク（`target="_blank"`）
- 現時点の掲載: 「株主総会 司会サービス LP（デザイン提案例）」1件
  - 「架空案件」の表記は「**デザイン提案例**」に変更（誠実さを保ちつつ前向きな枠にする）
- 増設予定（overall-design.md §13 並行タスク）: 飲食店デモサイト、個人事業主デモサイト → 3件で第1フェーズ完成
- レイアウト: 1件時は1カラム中央配置でも成立するデザインとする（件数が少ない状態を寂しく見せない）

### 8. 制作者について（About）

**目的**: 「フリーランスは不安」への信頼担保。RTBを人物情報として提示する。

- 顔写真必須（撮影予定 → 残タスク T-2。**写真が入るまで公開しない**）＋名前
- 自己紹介文の構成（順序固定）:
  1. **第1文でRTB**: 「IT企業で要件定義・設計などの上流工程とAWS運用を担当している現役エンジニアです」
  2. なぜ個人でやるのか: 「制作会社の品質を、中小企業や店舗が手の届く価格で提供したい」
  3. 人柄・姿勢: 専門用語を使わない対話、小さな相談歓迎
- 添える情報（確定）:
  - 対応時間: 平日夜・土日
  - 対応エリア: **対面での打ち合わせは首都圏のみ。その他の地域はオンラインで全国対応（要事前相談）**

### 9. よくあるご質問（FAQ）

**目的**: 相談直前の最後の不安を除去する。アコーディオン形式（6問）。

| Q | Aの要点 | 状態 |
|---|---------|------|
| 副業とのことですが、対応が遅くなりませんか？ | 連絡は原則24時間以内に返信。制作は週次報告で進捗を常に共有。納期は契約前に確定 | 確定 |
| 納期はどのくらいですか？ | 規模により変動。コーポレートサイトで目安◯〜◯週間。お急ぎの場合は事前相談 | **実数未定 → 残タスク T-3** |
| 何を用意すればいいですか？ | 何もなくて大丈夫。ヒアリングで整理するのが仕事。会社案内やメニュー表があればなお良し | 確定 |
| 公開後の更新はどうなりますか？ | WordPress構築なら自社で更新可能。スポットでの更新代行・保守も対応 | 確定 |
| 支払い方法・タイミングは？ | **着手金50%＋納品時50%**。銀行振込 | 確定（2026-07-05） |
| 直接会って相談できますか？ | **首都圏は対面での打ち合わせが可能**。その他の地域はオンライン（Zoom等）で全国対応（要事前相談） | 確定（2026-07-05） |

### 10. 無料相談（Contact）

**目的**: ゴール行動。入力負荷を最小化し、選択式中心にする。

- 見出し: 「無料相談」／リード文: 「『まだ何も決まっていない』段階のご相談も歓迎です。営業のご連絡は行いません。」
- フォーム項目:

| 項目 | 型 | 必須 | 備考 |
|------|----|------|------|
| お名前 | text | ○ | |
| メールアドレス | email | ○ | |
| ご相談内容 | select | ○ | コーポレートサイト／採用サイト／店舗サイト／機能追加／その他・まだ決まっていない |
| ご希望時期 | select | — | すぐに／1〜3ヶ月以内／未定 |
| 補足・ご質問 | textarea | — | 任意であることを明示（「空欄でも大丈夫です」） |

- 送信: Web3Forms（既存アクセスキー流用）。hidden の subject を「LPからの無料相談」に変更
- 送信後: 成功メッセージ「送信ありがとうございます。24時間以内にご返信します」（返信SLAをここでも明示）
- 背景: ダーク基調で締めのセクションとして強調（v1.0 踏襲）

### 11. フッター

- コピーライト＋（OQ-5 の結論に応じて）プライバシーポリシーへのリンク

## 4. アニメーション方針（GSAP / ScrollTrigger）

v1.0 の方針を踏襲（控えめ・上品・プログレッシブエンハンスメント・`prefers-reduced-motion` 対応）。

- Hero: label → コピー → CTA の順にフェードアップ（stagger）
- 悩みカード・理由カード・Flow: スクロール連動で順番にフェードアップ
- 過度な動きは §overall-design.md「避けたい印象」に反するため禁止

## 5. レスポンシブ方針

- モバイルファースト。PC は `@media screen and (min-width: 1080px)`
- 悩みカード: スマホ1列 → PC 2×2
- 理由カード: スマホ1列 → PC 2列＋1（または5列）
- Serviceカード: スマホ1列 → PC 2×2
- Flow: スマホ縦積み → PC 横並び5ステップ
- スマホ左右余白: `padding: 0 16px;`

## 6. クラス命名（ワークスペース規約準拠）

- レイアウト: `l_`／モジュール: `m_`／JSフック: `js_`／トグル状態: `.is-active`
- ページ固有セクション: `top_` プレフィックス（例: `top_pain`, `top_reason`, `top_flow`）
- 修飾クラスはブロック名を冠する（例: `m_btn__cta`）

## 7. 受け入れ基準

```
AC-1: 10秒テスト
Given 業界外の知人5人にHeroセクションのみを10秒見せる
When 「誰向けの・何のサービスか？」を質問する
Then 3人以上が「中小企業や店舗向けのWebサイト制作」と正答する

AC-2: 価格の透明性
Given 訪問者がページを最後まで読む
When 掲載情報を確認する
Then 「30万円〜」「機能追加5万円〜/件」「見積り無料」「追加費用は事前提示」が
     すべて明記されており、「準備中」の表記が1箇所もない

AC-3: CTA到達性
Given 任意のセクションを閲覧中
When 相談したくなる
Then ヘッダー常設CTAまたはセクション内CTAから1タップで #contact に到達できる

AC-4: フォーム送信
Given 必須項目（名前・メール・相談内容）を入力する
When 送信ボタンを押す
Then Web3Forms経由で送信され、成功メッセージが表示される
And 必須項目が未入力の場合はブラウザバリデーションで送信がブロックされる

AC-5: 表現ルール準拠
Given 実装済みのHTML全文
When 「AI」「Portfolio」「準備中」「お問い合わせ（CTAとして）」を検索する
Then いずれもヒットしない（AIは表現ルール§10-2、他は§2の削除方針による）

AC-6: レスポンシブ・アクセシビリティ
Given スマホ（375px）とPC（1280px）で表示する
When 全セクションを確認する
Then レイアウト崩れがなく、ハンバーガーメニューが aria-expanded 付きで開閉し、
     prefers-reduced-motion 有効時にアニメーションが無効化される
```
