# ポートフォリオサイト 全体設計

作成日: 2026-06-22

## 1. 背景・目的

フリーランスのWeb制作者として案件を受注するためのポートフォリオサイト。
クライアントに対して、人柄・スキル・実績・サービス内容を伝え、「この人に頼みたい」と感じてもらい、お問い合わせにつなげることを目的とする。

| 項目 | 内容 |
|------|------|
| 主な目的 | フリーランス受注 |
| ターゲット | 制作を依頼したいクライアント（企業・個人事業主など） |
| ゴール | サイトを見たクライアントがお問い合わせをする |

## 2. デザイン方針

**ミニマル・洗練（ホワイト × ブラック）**

- ベースカラー: オフホワイト（背景）× ブラック（テキスト・アクセント）
- 余白を大きく取り、情報を詰め込まず上品に見せる
- 細い罫線・控えめなアクセントで洗練された印象
- 高単価・感度の高いクライアントに好まれる落ち着いたトーン

フォント方針（実装時に詳細決定）:
- 和文: Noto Sans JP（本文）
- 見出しや英字に Noto Serif JP / 英字フォントを検討

## 3. サイト構成

ハイブリッド構成: **TOPページで全体を要約 → 詳細は個別ページへ誘導**

| ページ | ファイル | 役割 | 状態 |
|--------|---------|------|------|
| TOP | `index.html` | 全要素を簡潔に要約し、興味を引いて詳細へ誘導 | フル実装 |
| 実績一覧 | `works.html` | 制作実績を並べる | 枠組みのみ（実績ゼロ・「準備中」表示） |
| サービス | `service.html` | サービス内容・料金プラン | 枠組みのみ（内容未定・プレースホルダ） |

### TOPページのセクション構成（縦スクロール）

1. **Hero** — 名前・キャッチコピー（姿勢軸「丁寧なものづくりで、想いをカタチに。」）・職種・CTA（実績を見る / お問い合わせ）
2. **About** — 自己紹介（簡潔に）
3. **Skills（できること）** — 要件単位の提供メニュー: Webサイト制作 / LP制作 / SEO対策 / レスポンシブ対応 / WordPress構築 / 機能開発(AWS) / AI活用開発
4. **Works（抜粋）** — 実績の一部 →「もっと見る」で `works.html` へ
5. **Service（概要）** — サービス概要 →「詳しく見る」で `service.html` へ
6. **Contact** — お問い合わせフォーム（UIのみ）

共通ヘッダーのナビゲーションから各セクション・各ページへ移動できる。

## 4. 機能要件

| 機能 | 内容 | 今回の実装範囲 |
|------|------|--------------|
| お問い合わせフォーム | 入力欄・送信ボタン | **UIのみ**（送信処理は後回し）。将来 Formspree 等の外部サービス連携（方式A）を差し込める設計にする |
| ナビゲーション | 共通ヘッダー・各ページ/セクションへの移動 | 実装 |
| レスポンシブ対応 | スマホ・PC両対応 | 実装（モバイルファースト、`min-width: 1080px` のメディアクエリ） |

## 5. 技術スタック

| 項目 | 内容 |
|------|------|
| 言語 | HTML5 / CSS3 / JavaScript（Vanilla） |
| ビルドツール | なし（ブラウザで直接確認） |
| ホスティング | GitHub Pages（公開予定） |
| リポジトリ | https://github.com/antonion1215/portfolio |

CSS設計・クラス命名はワークスペース共通規約（`l_` / `m_` / `js_` プレフィックス、BEM風）に準拠する。

## 6. 今回の実装スコープ（明確化）

- **作り込む**: TOPページ全セクション、共通ヘッダー/フッター、お問い合わせフォームのUI、レスポンシブ
- **枠組みのみ**: 実績一覧ページ（実績コンテンツなし）、サービスページ（料金等の内容なし）
- **後回し**: お問い合わせフォームの実送信機能、実績データの追加、サービス内容の確定
